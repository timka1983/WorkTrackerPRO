
# Инструкция по размещению и использованию WorkTracker PRO

## 1. Подготовка репозитория (GitHub)
1. Создайте новый репозиторий на [GitHub](https://github.com/new).
2. Загрузите файлы проекта (через терминал или кнопкой "Upload files").
3. В настройках репозитория (**Settings -> Pages**) выберите источник **GitHub Actions**.

## 2. Как установить на телефон (Android/iOS)
Приложение работает как PWA (Progressive Web App).
1. Откройте ссылку на ваш опубликованный сайт в браузере **Chrome** (Android) или **Safari** (iOS).
2. **Android**: Нажмите на всплывающее уведомление "Установить" или выберите в меню (три точки) -> **"Установить приложение"**.
3. **iOS**: Нажмите кнопку "Поделиться" (квадрат со стрелкой вверх) -> **"На экран Домой"**.

## 3. Синхронизация данных
- Приложение использует **Supabase** для хранения данных в облаке.
- Не забудьте прописать ваши ключи `VITE_SUPABASE_URL` и `VITE_SUPABASE_ANON_KEY` в секретах GitHub (Settings -> Secrets -> Actions) для корректной сборки.

## 4. Оффлайн-режим
Благодаря Сервис-воркеру (`sw.js`), основные части приложения будут работать даже при временном отсутствии интернета. Данные синхронизируются, как только связь восстановится.

## 5. Блок отладки (Debug Block)
Блок отладки предназначен для технического обслуживания и исправления ошибок данных. Он доступен Супер-администраторам или работодателям, если для их организации включен "Режим отладки". Блок отображается в нижней части интерфейса работодателя.

### Элементы интерфейса и их функции:
1.  **Техническая сводка (Debug Info)**:
    *   `OrgID`: Уникальный идентификатор вашей организации в базе данных.
    *   `Users`: Количество загруженных сотрудников.
    *   `Logs`: Количество записей в журнале смен.
    *   `Active Shifts Map Keys`: Список ID сотрудников, у которых сейчас открыта смена.
    *   **Кнопка "Copy"**: Позволяет быстро скопировать эти данные для отправки разработчикам.

2.  **Инструменты исправления и обслуживания**:
    *   **Очистить кэш и перезагрузить**: Полностью очищает `localStorage` браузера. Используйте, если приложение ведет себя странно, данные "застряли" или интерфейс не обновляется.
    *   **Диагностика БД**: Запускает проверку целостности данных в Supabase и выводит отчет о найденных проблемах (например, некорректные связи или пустые записи).
    *   **Объединить дубликаты (по имени)**: Находит сотрудников с одинаковыми именами и объединяет их записи в один профиль, чтобы избежать путаницы в отчетах.
    *   **Исправить структуру БД (дубликаты позиций)**: Автоматически исправляет технические ошибки в справочнике должностей и функций.
    *   **Обновить данные из БД**: Принудительно загружает свежие данные с сервера, игнорируя локальный кэш. Помогает при расхождениях в данных на разных устройствах.
    *   **Принудительная очистка $**: Сбрасывает кэшированные финансовые расчеты, заставляя систему пересчитать зарплаты и бонусы заново на основе текущих логов.
    *   **Исправить дубликаты в БД**: Глобальный инструмент для удаления полностью идентичных записей в журнале смен, которые могли возникнуть при сбоях сети.
    *   **Проверить и удалить Base64 фото**: Оптимизирует базу данных, удаляя фотографии, сохраненные в тяжелом текстовом формате (Base64), что ускоряет работу приложения.

### Как пользоваться:
- **При "зависании" или ошибках интерфейса**: Нажмите "Очистить кэш и перезагрузить".
- **При расхождениях в зарплатах или отчетах**: Используйте "Обновить данные из БД" или "Принудительная очистка $".
- **При появлении лишних/повторяющихся сотрудников**: Нажмите "Объединить дубликаты".
- **Для обращения в техподдержку**: Нажмите "Copy" и вставьте полученный текст в сообщение администратору.

> **Внимание!** Используйте инструменты исправления только при возникновении реальных проблем с данными или по указанию технической поддержки.
